Overview

This page is the starting point for documenting a practical Jetson build instead of leaving the work as scattered experiments. The immediate goal is to get a clean baseline working: camera input, project structure, and a runtime flow that can grow into something more complete.

Over time, this page can hold milestone notes, architecture direction, debugging lessons, and progress snapshots.

Current Goal

  • Get the Jetson environment into a stable, usable state
  • Bring up camera input reliably
  • Establish a clean C++ project structure
  • Create a runtime loop that can evolve into a producer/consumer model

Architecture Direction

The current direction is to keep the project simple but structured. A useful split is:

  • input layer for camera and future sensors
  • runtime layer for orchestration and timing
  • processing layer for image or signal handling
  • output layer for logs, UI hooks, or future actions

The point is not to overdesign early. It is to avoid building something that becomes confusing the moment a second input or processing stage gets added.
